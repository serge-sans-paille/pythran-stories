Blah bvlah

  • Jun 23, 2023
  • Categories: mozilla 
#include <stdio.h>
__attribute__((noinline)) void foo() {
  char buffer[1024];
  if(scanf("%1023s", buffer) == 1)
    puts(buffer);
}

__attribute__((noinline)) int bar() {
  char buffer[128];
  if(scanf("%127s", buffer) == 1)
    puts(buffer);
  return 0;
}

int main(int argc, char ** argv) {
  if(argc > 1)
    foo();
  return bar();
}

Compiled with

clang -ftrivial-auto-var-init=pattern test.c -c -o/dev/null -O1 -g -Rpass-missed=annotation-remark

Gives too much information, we cannot handle everything automatically, let's use hotness information:

clang test.c -ftrivial-auto-var-init=pattern -fprofile-generate -O1 -o t

Do a few run, favouring calls to bar

seq 20 | xargs -I~ ./t

Merge profile information:

llvm-profdata merge default_*.profraw -o default.profdata

And feed that to clang again, using profile threashold information

clang -ftrivial-auto-var-init=pattern test.c -c -o/dev/null -O1 -g -Rpass-missed=annotation-remark -fdiagnostics-hotness-threshold=auto -fprofile-use=default.profdata -fdiagnostics-show-hotness
